<p>
    Part of <code>@Library('jenkins-pipeline-shared-libraries')</code>
</p>
<dl>
    <dt>
        <code>owaspDependencyCheck(scanDirs:List, tool: String, extraArgs:String):String</code>
    </dt>
    <dd>
        <p>
            Offers access to the OWASP Dependency Check. The generated reports (<code>--format 'ALL'</code>)
            dependency-check-junit.xml, dependency-check-report.csv, dependency-check-report.html,
            dependency-check-report.json and dependency-check-report.xml are written to the folder report
            (<code>--out report</code>) in the current workspace.
        </p>
        <ul>
            <li>
                <b>scan</b> (required) - The application directories to be scanned.
            </li>
            <li>
                <b>tool</b> (optional) - The name of the OWASP Dependency Check tool in the global configuration.
                Default: <code>'owasp-dependency-check-5.2.4'</code>.
            </li>
            <li>
                <b>extraArgs</b> (optional) - Additional arguments with which the the OWASP Dependency Check is called.
                For a full list of available arguments see <a href="https://jeremylong.github.io/DependencyCheck/dependency-check-cli/arguments.html">Command Line Arguments</a>.
                Some important arguments are:
                <ul>
                    <li>
                        <code>--enableExperimental</code> Enables the experimental analyzers. If not set, the analyzers
                        marked as experimental below will not be loaded or used.
                        See <a href="https://jeremylong.github.io/DependencyCheck/analyzers/index.html">File Type and Experimental Analyzers</a>.
                    </li>
                    <li>
                        <code>--exclude <pattern></code> The path patterns to exclude from the scan - this option can
                        be specified multiple times. This accepts Ant style path patterns (e.g. /exclude/).
                    </li>
                    <li>
                        <code>--failOnCVSS</code> If the score set between 0 and 10 the exit code from dependency-check
                        will indicate if a vulnerability with a CVSS score equal to or higher was identified.
                    </li>
                    <li>
                        <code>--suppression <files></code> The file paths to the suppression XML files; used to
                        suppress false positives. This can be specified more than once to utilize multiple suppression
                        files. The argument can be a local file path, a URL to a suppression file, or even a reference
                        to a file on the class path (https://github.com/jeremylong/DependencyCheck/issues/1878#issuecomment-487533799).
                        See: <a href="https://jeremylong.github.io/DependencyCheck/general/suppression.html">Suppressing False Positives</a>.
                    </li>
                    <li>
                        <code>--project <files></code> - The name of the project being scanned. This project name is
                        displayed as a heading in the report.
                    </li>
                </ul>
            </li>
        </ul>
        <p style="margin-bottom: 0em;">
            Example:
        </p>
        <pre style="margin-top: 0em; margin-left: 2em; color:#657383;"><code>
owaspDependencyCheck "app", "api", tool: "owasp-dependency-check-5.2.4", extraArgs: "--enableExperimental --suppression dependency-check-suppression.xml --failOnCVSS 7 --exclude exclude"
        </code></pre>
        <p>
            For further information see <a href="https://jeremylong.github.io/DependencyCheck/">dependency-check documentation</a>.
        </p>
    </dd>
</dl>
